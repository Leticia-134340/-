데이터 링크 계층

역할: 포인트 투 포인트 간 운송방식 결정
필요 요소: 연결 장비간 식별 주소(장비 식별), 오류 제어(데이터 손상 확인), 흐름 제어(데이터 전송 준비, 속도 조절)
장치: L2 SWITCH
LAN: 고속
CSMA/CD: 충돌 감지 및 처리
WAN: 안정
이더넷 프로토콜: 근거리 통신 환경에서 다수를 차지하는 2계층 프로토콜
반송파 감지, 충돌 검출, 반송파 접근 전에 매체가 사용 중인지 확인
문제점: 충돌 도메인 발생(여러 단말들이 서로 경쟁하여 충돌이 일어남)
해결방식
과거: 매체 공유(CSMA/CD 사용)
현재: 전용 포트 위주, 이더넷 스위치화, 고속화 지향(CSMA/CD 사용 안함)

문제

1 OSI 모델중 2계층 모델이 하는 역할에 대해서 설명하세요
역할: 장치 간 데이터 운송 방식 결정
프로토콜: LAN(이더넷), WAN(HDLC, PPP)

2 데이터 링크의 목적달성을 위한 필요 요소 3가지는 무엇인가요
장비 식별 , 오류 제어, 흐름 제어

3 LAN/WAN 의 대표 2계층 프로토콜의 명칭은 무엇인가요
LAN: 이더넷
WAN: HDLC

4 초기 이더넷 프로토콜의 문제점은 무엇인가요
CSMA/CD
충돌 도메인 발생
데이터 통신량이 많을 때 Collision domain 이발생-> 일시적으로(Jamming Time) 동안 모든 장치 통신 불가능

5 초기 이더넷 프로토콜이 사용하는 통신방법은 어떤것인가요
CSMA/CD

6 초기 이더넷 프로토콜의 문제점을 해결하기 위한 장치는
L25SWITCH


네트워크 계층

역할: 경로 탐색
장치: 라우터(rout+ er)
종류 
IP: 인터넷 프로토콜
ICMP: 인터넷 컴퓨터 메세지 프로토콜 정보통신용 프로토콜
ARP: 장치 간의 주소 학습(2계층을 위한 맥 주소 확보)
Routing Protocol: 길 안내 프로토콜

IP -> MAC -> 통신

문제

1 OSI 모델중 3계층 모델이 하는 역할은 무엇인가요
하는 일: 경로 탐색
약속: IP, ICMP, ARP, ROUTING PROTOCOL
장치: ROUTER

2 3계층 관련 장치는 무엇인가요
라우터

3 3계층 관련 프로토콜 종류는 어떤게 있나요
IP, ICMP, ARP, Routing protocol

4 ICMP 의 용도는?
통신유무 확인

5 IP 의 용도는?
논리적 주소를 통한 정치 식별

6 Routing 프로토콜의 용도는
경로탐색

전송 계층

계층이 올라갈 수록 하는 일이 많아져서 프로토콜이 복잡해짐
특히 4계층은 위아래 3개 계층의 연결을 담당하여 가장 중요

역할: 종단간 데이터의 신뢰성 보장
전송계층 프로토콜
TCP: 연결지향 프로토콜
신뢰적인 연결지향성 서비스
역할
신뢰성: 종단간 데이터 이상 유무
연결지향: 계속적인 데이터 교환을 위한 연결통로 설정

UDP: 비연결지향 프로토콜
구조가 단순한 전송계층 프로토콜. 빠른 데이터 전송에 적합
역할: 실시간 응용 및 멀티캐스팅 가능
빠른 요청과 응답이 필요한 실시간 응용에 적합
여러 다수 지점에 전송

문제

1 OSI 모델중 4계층 모델이 하는 역할에 대해서 서술하세요 
종단간 데이터 신뢰성 보장
송수신지간(End to End)의 신뢰성 있는 데이터 전송

2 4 계층 관련 프로토콜은 어떤게 있고 각각 특징을 서술하세요
TCP: 연결지향
데이터 신뢰도↑
속도↓
UDP: 비연결지향
데이터 신뢰도↓
속도↑
