네트워크 모델: 네트워크 구축 설계도

네트워크 모델 변화: 여러 네트워크 설계도 통일(표준 모델: OSI 7 Layer)

TCP/IP 모델: OSI 모델이 너무 세분화 되어 있어서 쓰는  모델(OSI보다 먼저 나오긴 했다)

물리 계층: 케이블 등을 통한 물리적 연결
데이터링크 계층: 데이터가 잘 전달 되도록 하는 것
네트워크 계층: 여러 장치들이 연결된 상태에서 수많은 길 중 목적지로의 가장 빠른 길을 찾는 것
전송 계층: 송수신지 간에 데이터가 잘 도착했는지 확인하는 것

문제

1. Network 는 무엇이며 무엇을 목적으로 하는가?
데이터 교환을 위해 여러 통신 장치를 연결한 통신망

2. Network 의 종류는 크게 ooo/ooo 로 나눌 수 있다
LAN: 비교적 가까운 거리의 통신망(근거리 통신망)
local
WAN: 넓은 범위의 통신망(광범위 통신망)
wide

3. LAN 의 특징에 대해서 서술하세요
범위: 제한적인 지리 범위(보통 한 건물, 인접한 건물)
속도: 높은 데이터 전송 속도
비용: 낮음
관리 용이성: 쉽고 간단
보안: 제한된 범위라서 외부에 비해 용이

비교적 가까운 형태로 연결한 통신

4. WAN의 특징에 대해서 서술하세요
범위: 지리적으로 넓은 구역(지역, 국가, 대륙 간에 연결을 지원)
기술: 다양한 기술 사용(유선, 무선)
인터넷 연결: 공용 인터넷을 많이 씀. 가상 연결망(vpn)을 통해 보안이 강화된 연걸 가능
비용: 광범위로 인한 운영 비용 높음
관리 복잡성: 대규모 네트워크이기 때문에 전문 지식과 복잡한 네트워크 관리 필요

비교적 먼 거리의 통신장치 간의 통신망

5. InterNet 이란
전세계적으로 연결된 컴퓨터 네트워크의 집합
공공, 개인 네트워크들이 상호 연결 됨
특징: 글로벌 연결
공개: 누구나 인터넷 연결 가능
예: 이메일, 파일 전송, 스트리밍, 클라우드 서비스

여러 종류의 통신망을 한 곳에 묶어놓은 거대 통신망

6. 프로토콜은 무엇인가?
네트워크 환경에서 데이터 통신을 위한 규칙이나 규약
규칙의 집합

통신을 하기 위해 사전에 정의한 규칙 

7. 네트워킹 모델의 목적은 무엇인가?
효율적인 네트워크 설계. 장애 발생 시 문제 원인 탐색 용이

통신망 구현을 위한 지침서
하자 발생시 참고하는 지침서
이기종간 통신을 위해 미리 정의해 놓은 표준 프로토콜 모델

8. TCP/IP 모델과 OSI 7 모델의 계층을 적고 각 계층별 역할을 적으세요
물리 계층: 장치간 물리적인 연결
데이터링크 계층: 장치간 데이터 운송방식 결정
네트워크 계층: 가장 빠른 경로 탐색
전송 계층: 송수신간의 데이터 신뢰성 확보
